# @areyouagentic/db

Prisma schema, migrations, and a typed singleton client for areyouagentic.com.

## What's in here

```
packages/db/
├── prisma/
│   ├── schema.prisma     # Models: AnalysisJob, Report, JobStatus
│   ├── seed.ts           # 3 example analyses (high/mid/low score)
│   └── migrations/       # Generated by `prisma migrate` (gitignored in dev)
└── src/
    ├── client.ts         # Singleton PrismaClient (dev-safe)
    ├── parse.ts          # parseReport() — Zod-validates JSONB columns
    ├── types.ts          # Typed Report view (narrows JSON fields)
    └── index.ts          # Public exports
```

The JSON columns (`findings`, `recommendations`, `evidence`) are typed via
Zod schemas in `@areyouagentic/shared`. Always go through `parseReport()`
when reading from the database — it validates the row against the current
schema and gives you a fully typed `Report`.

## Prerequisites

1. Postgres running locally — `docker compose up -d postgres` from the repo
   root brings up Postgres 16 on `localhost:5432`.
2. `DATABASE_URL` set in the repo-root `.env` (already wired up via
   `dotenv-cli` in the scripts below). Copy `.env.example` to `.env` if you
   haven't yet.
3. Dependencies installed — `pnpm install` from the repo root.

## Common commands

Run from `packages/db` (or `pnpm --filter @areyouagentic/db <script>` from
the root):

| Script                   | What it does                                                 |
| ------------------------ | ------------------------------------------------------------ |
| `pnpm db:generate`       | Regenerate the Prisma client from `schema.prisma`.           |
| `pnpm db:migrate`        | Create + apply a new dev migration. Prompts for a name.      |
| `pnpm db:migrate:deploy` | Apply pending migrations non-interactively (use in CI/prod). |
| `pnpm db:migrate:reset`  | Drop, recreate, re-migrate, and re-seed. **Destructive.**    |
| `pnpm db:push`           | Sync schema to DB without a migration. Prototype use only.   |
| `pnpm db:seed`           | Run `prisma/seed.ts`.                                        |
| `pnpm db:studio`         | Launch Prisma Studio at `localhost:5555`.                    |

## First-time setup

```bash
# from repo root
docker compose up -d postgres
pnpm install
pnpm --filter @areyouagentic/db db:generate
pnpm --filter @areyouagentic/db db:migrate    # name it e.g. "init"
pnpm --filter @areyouagentic/db db:seed
```

After that, opening Prisma Studio is a quick sanity check:

```bash
pnpm --filter @areyouagentic/db db:studio
```

## Adding a migration

1. Edit `prisma/schema.prisma`.
2. Run `pnpm db:migrate` and pick a descriptive name
   (`add-user-table`, `report-add-final-url`, etc.).
3. Commit the generated SQL files under `prisma/migrations/` along with the
   schema change. Never edit applied migrations — write a new one.

## Using the client

```ts
import { prisma, parseReport } from '@areyouagentic/db';

const row = await prisma.report.findUniqueOrThrow({ where: { id } });
const report = parseReport(row); // typed Report with parsed JSONB
```

For writes, build the JSON payloads with the Zod schemas from
`@areyouagentic/shared` and pass them straight in:

```ts
import { findingsSchema, recommendationsSchema, evidenceSchema } from '@areyouagentic/shared';
import { prisma, Prisma } from '@areyouagentic/db';

await prisma.report.create({
  data: {
    // ...scores
    findings: findingsSchema.parse(findings) as unknown as Prisma.InputJsonValue,
    recommendations: recommendationsSchema.parse(recs) as unknown as Prisma.InputJsonValue,
    evidence: evidenceSchema.parse(evidence) as unknown as Prisma.InputJsonValue,
  },
});
```

The `as unknown as Prisma.InputJsonValue` cast is just to satisfy Prisma's
strict JSON type — the runtime value is already validated.

## Resetting the dev database

```bash
pnpm --filter @areyouagentic/db db:migrate:reset
```

This drops the schema, re-applies every migration from scratch, and runs
the seed. Use it freely in dev; never run it against shared environments.
